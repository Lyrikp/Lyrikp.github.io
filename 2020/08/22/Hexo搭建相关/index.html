Hexo搭建相关
Aug 22 2020

搭建之后

前提条件：在将网站搭建成功之后，我们应该这样子更新网站

新建文章hexo new

简写：hexo new命令可以简写为hexo n

简单操作

新建一个名称为“miao”的文章

$ hexo new miao
  • 新建的文章在source - _posts文件夹
  • 如果要更改文章所在的分页，需要前往文章本身修改开头（之后会讲）

邪门操作：你也可以直接去source/_posts文件夹直接新建一个md格式的文章

详细表述

  • <title> 新建的文章标题 使用时不加<>
    • 空格需要使用/ 或者" "处理
    • 文件名miao miao表示为 miao/ miao 或者 "miao miao"
  • [layout] 新建的类型，非必填项
    • 文章post 不填时默认的类型为这个
    • 分页page 新建网站的分页
    • 草稿draft 草稿不会显示在发布的地方，未完成的文章可以先放在这里，然后通过这个代码来直接发布

hexo new命令的全部表述为

$ hexo new [layout] <title>

编辑文章

在新建出来的md文件里面直接修改就可以了。
md文件即_markdown_格式编码的文件。可以通过简单的符号处理生成好看的笔记与文章格式，如果您处理过_LaTax_文件，那么_markdown_语法上手会相当幼儿园。
这里暂时不叙述文件的编写方式。

预览

在进行网站更改之后，我们在上传之前或许需要提前预览一下
hexo server命令可以在本体服务器上进行预览
hexo s为这个命令的简写

上传

完成之后进行冗杂数据清除和复写上传

hexo clean && hexo g -d

预览

在某一个网站上进行预览

hexo s
